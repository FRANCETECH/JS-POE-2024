Les erreurs

La gestion des erreurs est une partie crucial de la programmation, elle permet de s'assurer que votre code continue de s'executer même si des problèmes inattendus se produisent.

Try/Catch

Le bloc try/catch permet de capturer une erreur sans interrompre le reste du code

Throw

Permet de générer des erreurs personnalisées

Finally

Finally fonctionne avec try/catch pour executer du code indépendamment du fait qu'une erreur soit capturée.

const error = new Error("Erreur de type Error") // fonction constructeur console.log(error.message) console.log(error.name) console.log(error.stack) // throw throw new Error("Oooooops") new ReferenceError("Erreur de reference") new SyntaxError("Erreur de syntaxte") new TypeError("Erreur de type") consol.log("dsfjidke") // try/catch function echec() { try { throw new Error("ahhahha") console.log("hehehhe") } catch(e) { console.error(`Erreur capturée : ${e}`) } finally { console.log("zehahaha") } } // si on ne capture l'erreur le programme s'arrete function echec2() { throw new Error("ahhahha") console.log("hehehhe") } echec2() // stop le programme console.log("fin du script") // si opération async, le catch doit être dans le callback function echec3() { setTimeout(() => { try { throw new Error("Ops") } catch (e) { console.log("Erreur: " + e) } }, 2000); } function echec4() { try { setTimeout(() => { throw new Error("Ops") }, 2000); } catch (e) { console.log("Erreur: " + e) } }
